Chapter 3

🏞️ Delta Lake — The Living Lake

Data lands in Delta Lake — an open-source storage protocol that turns cloud files into ACID-compliant, time-travel-capable tables. Think of it as the lake at the heart of our park: everything flows here. Data moves through the Medallion Architecture — Bronze → Silver → Gold. Hover each tier for syntax.

DELTA LAKE — ACID · VERSIONED · STREAMING
🥉
Bronze — Raw Ingestion
Exact copy of source data. Schema as-is. Append-only. Audit trail preserved.

🥉 Bronze Layer — Raw Data

Ingest from all sources with minimal transformation. Keep source fidelity.

-- Create bronze streaming table CREATE OR REFRESH STREAMING TABLE bronze.raw_orders AS SELECT current_timestamp() AS _ingested_at, * FROM STREAM READ_FILES( 'abfss://raw@store.dfs.core.windows.net/orders/', format => 'json', schemaHints => 'order_id BIGINT, amount DOUBLE' ); -- Delta property: append-only recommended ALTER TABLE bronze.raw_orders SET TBLPROPERTIES ('delta.appendOnly' = 'true');
🥈
Silver — Cleaned & Conformed
Validated, deduplicated, standardised. Business-aligned schemas. Joined with reference data.

🥈 Silver Layer — Conformed Data

Apply business rules, deduplicate, enforce schema, join dimensions.

-- Silver materialized view CREATE OR REFRESH MATERIALIZED VIEW silver.orders AS SELECT o.order_id, o.customer_id, c.customer_name, o.amount, CAST(o.order_date AS DATE) AS order_date, UPPER(o.status) AS status FROM STREAM(bronze.raw_orders) o JOIN silver.customers c ON o.customer_id = c.id WHERE o.amount > 0 AND o.order_id IS NOT NULL; -- Constraints ALTER TABLE silver.orders ADD CONSTRAINT valid_amount CHECK (amount > 0);
🥇
Gold — Business Aggregates
Curated, aggregated, BI-ready. Serves dashboards, ML models, and data apps.

🥇 Gold Layer — BI & ML Ready

Pre-aggregated fact tables, wide tables, feature stores. Optimised for read performance.

-- Gold aggregate table CREATE OR REFRESH MATERIALIZED VIEW gold.daily_revenue AS SELECT order_date, customer_name, SUM(amount) AS total_revenue, COUNT(DISTINCT order_id) AS order_count, AVG(amount) AS avg_order_value FROM silver.orders WHERE status = 'COMPLETED' GROUP BY order_date, customer_name; -- Optimize with liquid clustering OPTIMIZE gold.daily_revenue ZORDER BY (order_date); -- Auto vacuum VACUUM gold.daily_revenue RETAIN 168 HOURS;
⏰
Time Travel
Query any past version

⏰ Delta Time Travel

Query historical table versions using timestamp or version number.

-- By version number SELECT * FROM silver.orders VERSION AS OF 10; -- By timestamp SELECT * FROM silver.orders TIMESTAMP AS OF '2024-01-15 00:00:00'; -- Restore to previous version RESTORE TABLE silver.orders TO VERSION AS OF 5; -- View history DESCRIBE HISTORY silver.orders;
🔀
MERGE / Upsert
SCD Type 1 & 2 support

🔀 MERGE — Upsert Operation

Atomically insert, update, or delete rows based on matching conditions.

MERGE INTO silver.customers AS target USING bronze.customer_updates AS source ON target.customer_id = source.customer_id WHEN MATCHED AND source.is_deleted = TRUE THEN DELETE WHEN MATCHED THEN UPDATE SET target.email = source.email, target.updated_at = current_timestamp() WHEN NOT MATCHED THEN INSERT (customer_id, email, created_at) VALUES (source.customer_id, source.email, current_timestamp());
📊
Change Data Feed
CDC from Delta tables

📊 Delta Change Data Feed (CDF)

Propagate only changed rows downstream for efficient incremental pipelines.

-- Enable CDF ALTER TABLE silver.orders SET TBLPROPERTIES ( 'delta.enableChangeDataFeed' = 'true' ); -- Read changes df_changes = (spark.readStream .format("delta") .option("readChangeFeed", "true") .option("startingVersion", 5) .table("silver.orders")) df_changes.filter( col("_change_type").isin(["insert","update_postimage"]) ).writeStream.format("delta") \ .toTable("gold.order_changes")
💧
Liquid Clustering
Auto-optimised layout

💧 Liquid Clustering

Replaces static partitioning. Databricks auto-clusters data for best query performance.

-- Create with liquid clustering CREATE TABLE gold.sales_facts CLUSTER BY (sale_date, region) AS SELECT * FROM silver.orders; -- Trigger clustering manually OPTIMIZE gold.sales_facts; -- Delta data skipping applies automatically -- ZOrdering is now handled by liquid clustering -- No manual ZORDER needed
PySpark / SQL
# Delta Lake table operations from delta.tables import DeltaTable # Read a Delta table df = spark.read.table("catalog.silver.orders") # Describe table details spark.sql("DESCRIBE DETAIL silver.orders").show() # Table statistics spark.sql("ANALYZE TABLE silver.orders COMPUTE STATISTICS") # Schema evolution: add a column spark.sql("ALTER TABLE silver.orders ADD COLUMN discount DOUBLE") # Convert Parquet to Delta (one-time migration) spark.sql("""CONVERT TO DELTA parquet.`abfss://data@store.dfs.core.windows.net/legacy/orders/`""")

🏛️ Unity Catalog — The Park Registry

Every park needs a registry: who owns which plot, who can enter, what's on each parcel. Unity Catalog is that registry — governing all data & AI assets across every workspace with a 3-level namespace: Catalog → Schema → Table. Hover any node to see how permissions work.

🏛️ Metastore (account-level)

🏛️ Unity Catalog Metastore

One metastore per region. All catalogs, schemas, tables, volumes, models, functions registered here. Automatically enabled for workspaces created after Nov 2023.

-- View metastore info SELECT * FROM system.information_schema.metastores; -- Grant workspace access GRANT USE CATALOG ON METASTORE TO `workspace_admins`;
📁 main (catalog)

📁 Catalog — Top-Level Namespace

Catalogs isolate domains: prod, dev, analytics, external. Map to business units or environments.

-- Create catalog CREATE CATALOG analytics_prod COMMENT 'Production analytics data'; -- Grant usage GRANT USE CATALOG ON CATALOG analytics_prod TO `data_analysts`; -- Three-level namespace SELECT * FROM analytics_prod.silver.orders; -- ^catalog ^schema ^table
📁 dev (catalog)

📁 Dev Catalog

Isolated dev environment. Engineers experiment without touching prod data.

CREATE CATALOG dev COMMENT 'Development & experimentation'; GRANT ALL PRIVILEGES ON CATALOG dev TO `data_engineers`; -- Clone prod table for dev testing CREATE TABLE dev.silver.orders CLONE analytics_prod.silver.orders;
📁 external (catalog)

📁 External Data Catalog

Governs external tables and volumes stored in your own ADLS. Unity Catalog manages governance only.

-- Create external location CREATE EXTERNAL LOCATION ext_storage URL 'abfss://ext@store.dfs.core.windows.net/' WITH (STORAGE CREDENTIAL main_cred); -- External table CREATE TABLE external.bronze.raw_data LOCATION 'abfss://ext@store.dfs.core.windows.net/raw/' USING DELTA;
