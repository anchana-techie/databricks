📥 Ingestion Methods — How Data Lands in the Lakehouse
🔄 Auto Loader
StreamingIncremental
  • cloudFiles() format source
  • File notification mode (SQS / Event Grid)
  • Directory listing mode (fallback)
  • Auto schema inference & evolution
  • Checkpoint-based exactly-once semantics
  • Scales to billions of files per directory
  • Supports JSON, CSV, Parquet, Avro, ORC, text
  • Writes via writeStream to Delta table
  • rescuedDataColumn for schema mismatches

✅ Best for: S3/ADLS/GCS → Bronze continuous ingestion

📋 COPY INTO
IdempotentBatch SQL
  • SQL command: COPY INTO table FROM path
  • Tracks ingested files in Delta transaction log
  • Skips already-loaded files automatically
  • FORMAT options: CSV, JSON, Parquet, Avro, ORC
  • PATTERN for glob-based file filtering
  • COPY_OPTIONS (header, delimiter, etc.)
  • No checkpoint management required
  • Good for ≤ millions of files per dir
  • Can run on SQL Warehouses (serverless)

✅ Best for: periodic batch / ad-hoc file loads

🏗️ CTAS / INSERT / MERGE
Full RefreshSQL DDL
  • CREATE TABLE AS SELECT (CTAS)
  • CREATE OR REPLACE TABLE AS SELECT
  • INSERT INTO — append rows
  • INSERT OVERWRITE — replace partition
  • MERGE INTO — upsert / SCD Type 1
  • Dynamic partition overwrite mode
  • Used in DLT SQL pipelines declaratively
  • SELECT from external tables / views
  • Supports column mapping & type evolution

✅ Best for: transformations, full reloads, upserts

🌊 Structured Streaming
Micro-batchContinuous
  • spark.readStream / writeStream API
  • Trigger.ProcessingTime("5 minutes")
  • Trigger.AvailableNow() — batch-like run
  • Trigger.Continuous("1 second")
  • Sources: Kafka, Kinesis, EventHub, Delta
  • Watermarking for late-arriving data
  • Stateful aggregations (window functions)
  • foreachBatch for Delta merge/upsert
  • Checkpointing to DBFS/cloud storage

✅ Best for: real-time event stream processing

🔌 JDBC / Lakehouse Federation
BatchFederated
  • spark.read.jdbc(url, table, props)
  • Partition column for parallel reads
  • numPartitions for parallelism
  • predicates push-down to source DB
  • External tables on S3/ADLS (Hive)
  • Unity Catalog foreign tables
  • Lakehouse Federation (query without copy)
  • Connections: PostgreSQL, MySQL, Redshift, etc.

✅ Best for: DB extracts & federated querying

📡 DLT Internal Sources
DeclarativeCDC
  • dlt.read("table_name") — batch read
  • dlt.read_stream("table") — streaming
  • dlt.apply_changes() — CDC upsert
  • SQL: APPLY CHANGES INTO target FROM src
  • KEYS clause for primary key definition
  • SEQUENCE BY for ordering changes
  • SCD Type 1 (default) & SCD Type 2 support
  • Auto dependency graph resolution
  • Backfill & full refresh on demand

✅ Best for: pipeline-internal CDC & SCD handling

⚡ Spark Declarative Pipelines — Delta Live Tables (DLT)
🥉 Bronze Layer
Raw Ingestion
  • @dlt.table (streaming source)
  • cloudFiles / Kafka readStream
  • No transformations applied
  • Append-only, schema raw
  • _rescued_data column
  • Immutable audit trail
→
🥈 Silver Layer
Cleaned / Conformed
  • @dlt.table (batch or stream)
  • MERGE / apply_changes() CDC
  • Type casting & date parsing
  • Deduplication logic
  • @dlt.expect() quality rules
  • Null handling & filters
→
🥇 Gold Layer
Business Aggregates
  • @dlt.table (materialized view)
  • Multi-table joins
  • Business KPI aggregations
  • CTAS / SQL SELECT patterns
  • Slowly Changing Dimensions
  • Serving-layer optimized
🛡️ Data Quality Controls
  • @dlt.expect("name", condition)
  • expect_or_drop — filter invalid rows
  • expect_or_fail — halt on violation
  • Quarantine table pattern
  • Quality metrics in Event Log
  • system.event_log table queries
🔁 CDC — apply_changes()
  • APPLY CHANGES INTO target
  • FROM source KEYS (pk_col)
  • SEQUENCE BY event_timestamp
  • SCD Type 1 — upsert (default)
  • SCD Type 2 — history rows
  • STORED AS {SCD_TYPE_2} syntax

Compute plane — Spark execution engine
🖥️ Cluster Layer — Compute Types & Configuration
⚙️ All-Purpose Cluster
Interactive
  • Multi-user shared compute
  • Notebooks & ad-hoc SQL
  • Manual start/stop control
  • Auto-termination on idle
  • Persistent across jobs
  • Standard / Premium tiers
  • Autoscaling: min→max workers
  • High cost if left running
🏭 Job Cluster
Ephemeral
  • Spun up per job run only
  • Terminated on completion
  • No idle cost — most efficient
  • Defined in Job JSON config
  • new_cluster per task option
  • Spot + On-demand mix
  • Spot fallback policy config
  • Instance pools for fast start
⚡ Serverless Compute
Managed
  • No cluster config required
  • ~2s cold start time
  • Databricks-managed VPC
  • SQL Warehouses (BI queries)
  • Serverless Jobs compute
  • Serverless DLT pipelines
  • Per-second billing granularity
  • No Spark UI (limited debug)
🌊 DLT Pipeline Cluster
Auto-managed
  • Auto-provisioned by DLT engine
  • Enhanced autoscaling (DLT-specific)
  • Driver + Worker node split
  • One cluster per pipeline
  • Restart on failure policy
  • Custom spark.conf allowed
  • Cannot be used for other tasks
  • Photon optional on DLT
🔧 Node & Runtime Config
  • Driver: DAG coordination, result collection
  • Worker nodes: parallel task execution
  • Memory-optimized (r-family AWS / E-series Azure)
  • Compute-optimized (c-family / F-series)
  • GPU instances for ML workloads
  • Photon Engine: vectorized C++ execution
  • Delta Cache: SSD-cached Parquet on workers
  • DBR version pinning (14.x, 15.x, etc.)
📈 Autoscaling & Policies
  • Standard autoscaling — queue-based
  • Enhanced autoscaling — predictive scale-down
  • Cluster policies — enforce org-wide configs
  • Instance pools — pre-warmed idle VMs
  • Auto-termination: idle timeout (minutes)
  • Init scripts — custom env on node boot
  • Libraries: PyPI, Maven, DBFS wheel
  • Spot instance bid strategy config
🔐 Access & Security
  • Single-user mode (UC enforced)
  • Shared mode (multi-user isolation)
  • No isolation shared (legacy)
  • DBR ML — MLflow, scikit-learn, TF
  • Unity Catalog data access control
  • Credential passthrough (legacy AAD)
  • Network security groups / private link
  • Cluster-scoped env variables for secrets
